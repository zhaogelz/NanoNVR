**本方案特别适合手头有闲置旧电脑，又需要装监控的人群。**

## 📌 诞生背景

**想要装个监控，但这几个痛点劝退了很多人：**
1.  **室外储存卡不安全：** 装在室外的监控，如果只插储存卡，不仅因为风吹日晒坏得极快，而且一旦遇到坏人，直接连卡一起拿走毁尸灭迹，极不安全。
2.  **传统 NVR 太贵：** 购买专用的 NVR 录像机 + 监控级硬盘的组合成本实在高昂。

因此，我写了这个开源项目，**旨在将任何一台吃灰的闲置旧电脑 / 笔记本**，一键转化为稳定可靠的**局域网监控录像机（NVR）**！。

### 🛒 硬件选购建议（极简路线）
仅需购买**一台普通的监控摄像头**，接上网线或 Wi-Fi连接即可（前提是网络不能太差）。
*   **关键要求**：购买前问下客服，确认摄像头支持**主/子码流**，且必须支持 **H.265 编码**（H.265 极其节省空间！非常关键。例如我自购测试的型号是 `TL-IPC682F-A`）。

## 🛡️ 核心亮点：为旧笔记本“极限护盘”

旧电脑或笔记本的固态硬盘(SSD)有写入寿命限制，如果像普通录像机一样持续暴力碎写，极易导致写废或爆盘。因此，NanoNVR 在底层做了大量专属的**护盘优化**：

1. **🛡️ 内存级大缓冲（减少小文件高频 I/O）**：
   在底层的 FFmpeg 流抓取参数中注入 `-buffer_size 10M`。它会利用笔记本充裕的内存，先将视频流积攒在内存中，再进行**大块连续写入**。大幅减少针对固态闪存的碎小高频读写，极大延长硬盘寿命。
2. **🗑️ 智能高低水位批量清理（杜绝爆盘与反复横跳）**：
   设定最大硬盘限额后（如 150GB 代表"高水位"），一旦触发红线，程序会**智能计算出"低水位"目标**（一次性释放约 5GB 的安全空间），跳过回收站直接批量物理抹除最老的视频。彻底杜绝了程序在警戒线附近"删一个写一个"、高频刷新文件分区表的折磨盘面行为。
3. **💤 惰性轮询机制（降低系统开销）**：
   抛弃“狂躁”的实时硬盘扫描，采用 5 分钟周期的“惰性轮询”来监控容量，极大地**降低了对磁盘文件分配表（目录树）的读取频率与发热量**。
4. **📦 规整的顺序切片写入**：
   严格将视频切片为 15 分钟的独立 `.ts` 序列，并在每天午夜自动建立 `年-月-日` 新归档文件夹。确保硬盘进行的是健康的**连续顺序写入**，即使断电也不会导致整个动辄几十 G的大文件报废。

> **💡 关于 内存缓冲(`-buffer_size`) 与 虚拟内存盘(RAMDisk)？**  
> 骨灰玩家喜欢在系统里划出一块"RAMDisk"把录像全存在内存，15分钟后再移到硬盘以实现 0 写放大。但这**需要安装第三方底层驱动和管理员提权**。为了坚持"免配置、双击即用"的傻瓜式理念，我们选用了无侵入的传输层内存缓冲方案，这已经足以大幅护盘。

## 🚀 极简使用方法

1. **下载运行**：从上述开源地址下载最新的 `.exe` 应用程序文件，放到旧电脑上双击运行。
2. **填入视频流**：在主界面的"RTSP 流地址"输入框中，填入您摄像头的主或子码流地址（例如 `rtsp://admin:password@192.168.1.100/stream1`）。
3. **划定空间**：在"空间限制(GB)"框填入想分配的最高硬盘限额数值（填 `150` 则录像占用绝不会超过 150GB）。
4. **一键录播**：点击**"▶ 启动录制"**按钮。程序会自动保存参数自动录制。

> **⚠️ 关于播放与异地使用的特别说明：**
> *   **务必使用专业播放器**：为了最大化节省硬盘空间，NanoNVR 保存的视频均为**未转码的原始流**。普通的系统自带播放器大概率无法发声或出画面，您**必须使用 VLC 播放器**（或任意支持 H.265 的播放器）来观看。
> *   **异地远程录制**：上述方案仅适合本地局域网内录制。如果您要异地录制，需要做个内网穿透，将码流挂到公网。我自己的做法是：由于我自己有云服务器，在老家路由器里做个 **`frp` 内网穿透** ，这样就能极其轻松地实现异地监控查看了。

### 📂 录像文件保存在哪里？

启动录制后，录像文件会直接**就近保存**在 **NanoNVR.exe 所在的同级目录**下！它们会自动按天建立文件夹，按时间切片（直接使用 VLC 或 PotPlayer 等播放器就能直接打开回看）：

```text
NanoNVR.exe 所在目录/
├── NanoNVR.exe           # 主程序
├── config.json           # 自动保存的配置参数
├── 2026-03-01/           # 【按日期自动归档文件夹】
│   ├── 08_00_00.ts       # 08:00 开始的 15 分钟录像切片
│   ├── 08_15_00.ts       # 08:15 开始的 15 分钟录像切片
│   └── ...
└── ...
```

## 🛠️ 打包编译指南 (供开发者)

如果对代码进行了二次修改，需要重新打包给其他纯净系统使用：

1. 确保当前环境已安装 Python 与 `pyinstaller`。
2. 确保 `ffmpeg.exe` 位于 `recorder.py` 旁边。
3. 双击运行 `build_exe.bat` 进行自动编译封装。
4. 编译完成会自动打开 `dist` 目录，提取生成的独立执行文件即可。
